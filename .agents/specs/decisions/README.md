## Criando um ADR

1. Copie `template.md` para `NNNN-titulo.md`.
2. Registre contexto, opcoes, decisao e consequencias.
3. Use `Proposed` enquanto estiver em discussao.
4. Depois da decisao, altere o status e atualize este indice.
5. Nao reescreva uma decisao aceita; crie outro ADR que a substitua.

## Status

- `Proposed`: em discussao.
- `Accepted`: adotado pelo projeto.
- `Rejected`: avaliado e nao adotado.
- `Deprecated`: deixou de ser recomendado.
- `Superseded`: substituido por outro ADR.
